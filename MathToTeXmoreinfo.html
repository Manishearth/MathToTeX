What it does

This script converts most "typed" math into properly formed $\LaTeX$. For example, (m_0)/(sqrt(1-(v^2)/(c^2))) becomes \frac{m_0}{\sqrt{1-\frac{v^2}{c^2}}}, which can be displayed as $\frac{m_0}{\sqrt{1-\frac{v^2}{c^2}}}$

The parser does a wide range of things, including:

  • Identifying fractions, superscripts, subscripts, and square roots properly and converting them to $\LaTeX$
  • Converting many different Unicode characters into their $\LaTeX$ equivalents
  • Converting greek letters(both Unicode and typed out) and trig functions into $\LaTeX$
  • Attempting to deal with ambiguous cases like "1/a+b"
  • Other stuff which I can't think of
You can see some more examples here. Copy-paste them into the converter and try them out!

Note: there is a known issue with square roots, where stuff like 1/sqrt(2) is not properly converted.

How it works

Basically this script breaks the input recursively into parenthesis-delimited objects. So, we can have an object tree that depends upon the nested parentheses. To deal with ambiguities, it normalizes certain input before it puts it into the parser. For example, 1/a+b is converted into (1)/(a+b). Then, the object can be "compiled", where the object tree is traversed and recursively assembled into $\LaTeX$. Finally, there are conversion methods which can be applied to do replacements of Greek letters, etc. There is more documentation embedded in the script
